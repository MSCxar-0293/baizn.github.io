发表于 2015-11-10

SPA单页面WEB应用

什么是SPA

单页面Web应用（single-page application 简称SPA）是一种特殊的Web应用。它将所有活动局限于一个Web页面中，仅在该Web页面初始化时加载相应的HTML，CSS和JavaScript。一旦页面加载完成，SPA不会因为用户的操作而进行页面的重新加载或跳转，而是利用JS动态的变换HTML的内容，从而实现用户与UI之间的交互。由于避免了页面的重新加载，SPA可以提供较为流畅的用户体验。

优缺点

SPA的出现是富客户端发展的必然结果，但该技术也是有些局限的。

优点

  • 良好的交互体验：用户不需要重新刷新页面，获取数据是通过Ajax异步获取，页面显示流畅
  • 前后端分离：SPA可以和RESTful规约一起使用，通过REST API提供接口数据，并使用Ajax异步获取，有助于分离客户端和服务器端的工作
  • 减轻服务器压力：服务器只用提供数据，不用管展示逻辑和页面生成
  • 共用一套后台代码：不用修改后端程序就可以同时用于Web，手机和平板等多种客户端

缺点

  • SEO难度较高：由于所有的内容都在一个页面中动态替换显示，所以在SEO上有劣势
  • 浏览器自身的前进后退管理：由于SPA在一个页面中显示所有内容，所以不能使用浏览器的进行后退功能，所有的页面切换需要自己建立堆栈管理，可以使用HTML5的history API来实现
  • 初次加载比较慢：为实现SPA功能及显示效果，需要在加载页面时候将JS，CSS统一加载，部分页面可以在需要的时候加载。所以必须对JS和CSS代码进行合并压缩处理

IE7+/firefox行为一致，不包括position=absolute的元素，且不管页面缩放的全文高度；chrome计算宽度和高度时候同时包含了position＝absolute的元素，且当页面缩放时，当全文高度／宽度小于可视区域的高度和宽度时候，计算的结果是可视区域的高度和宽度。

  • window.screen.widht

当页面不缩放时候，IE，firefox和chrome都获得真实的屏幕分辨率；当页面缩放时，chrome返回的值不变，IE7+和firefox返回屏幕分辨率/缩放比例。

网页可视区域的高和宽

  • 宽度：document.body.clientWidth
  • 高度：document.body.clientHeight

该高度和宽度不包含浏览器右侧和底部的滚动条，也不包含body的margin。

高度 ＝ Min(页面高度，可视高度)

宽度 ＝ Max(页面宽度，可视宽度)

页面缩放后，高度不变，宽度＝（原始宽度／缩放比例）。

网页被卷进去的高和宽

  • 宽度：window.pageXOffset; document.documentElement.scrollLeft
  • 高度：window.pageYOffset; document.documentElement.scrollTop

ScrollHeight / OffsetHeight / ClientHeight的区别

  • offsetHeight为元素的高度，被挡住部分同样计算在内；其父元素的overflow属性对此没有影响，自身的overflow对此也没影响
  • scrollHeight为元素的高度，被挡住的部分同样计算在内；其父元素的overflow对此没影响，受自身overflow影响
  • clientHeight为元素的高度，被挡住部分同样计算在内；其父元素的overflow对此没影响，受自身overflow的影响

IE系列浏览器的滚动条的宽度和高度是固定的16px，不受缩放影响；firefox和chrome的滚动条在100%的时候是16px，当缩放时，该宽度和高度发生了变化，为16/随风比例 px。

元素的offsetTop

  • 对于position=absolute的元素，offsetTop为其相对于其祖先元素中最近的一个position=relative或position=absolute的元素的偏移
  • 对于position=absolute的元素，offsetTop为其top属性值指定的值，如果该属性为空，则与position!=absolute时候的算法相同

元素的scrollTop

scrollTop是指某一元素内部的元素被卷进去的高度，而不是该元素本身被卷进去的高度。只有当该元素的overflow=auto或scroll时候，该值才有意义。

ECMAScript自动分号插入

必须使用分号终止某些ECMAScript语句（空语句，变量声明语句，表达式语句，do-while语句，continue语句，break语句，return语句，throw语句）。这些分号总是明确的显示在源文本中，但是有时为了方便，某些情况下可以在源文本中将这些分号省略。描述这种情况的时候会说给源代码的token流自动插入分号。

自动分号插入规则

分号插入有三个基本规则。

  • 左到右解析程序，当遇到一个不符合任何文法产生式的token，那么只满足下面条件之一就在违规token前面自动插入分号

    至少一个LineTerminator分割了违规token和前一个token
    违规token式｝

  • 左到右解析程序，token输入流已经结束，当解析器无法将输入token流解析成单个完整ECMAScript程序，那么就在输入流的结束位置自动插入分号
  • 左到右解析程序，遇到一个某些文法产生式允许的token，但是此产生式是受限产生式，受限产生式后紧跟在no LineTerminator here后的第一个终结符或非终结符的token叫做受限token，当至少一个LineTerminator分割了受限的token和前一个token，那么就在受限token前面自动插入分号

上述规则有一个前提条件：如果插入分号后解析结果是空语句，或如果插入分号后它成为for语句头部的两个分号之一，那么不会自动插入分号。

这些受限式产生的实际效果如下：

  • 当遇到的++ 或 – token将要被解析器当作一个后缀运算符，并且至少有一个LineTerminator出现在++ 或 – token和它之前的token之间，那么在++ 或 – token前面自动插入一个分号
  • 当遇到continue，break，return，throw token，并且在下一个token前面遇到LineTerminator，那么在continue，break，return，throw token后面自动插入一个分号

这些规则对实际编程中的影响如下：

  • 后缀运算符++ 或 – 和它的操作数应该在同一行
  • return或throw语句的表达式开始位置应该和return或throw token在同一行
  • break或continue语句的标示符应该和break和continue token在同一行

使用箭头函数需要注意的几点：

  • 函数体内的this对象，是定义时所在的对象，而不是使用时所在的对象
  • 不可以当作构造函数，即箭头函数不可以使用new命令
  • 不可以使用arguments对象，该对象在箭头函数体内不存在，可以使用rest参数达到相同的目的
  • 不可以使用yield命令，箭头函数不是Generator函数

this对象的指向是可变的，但在箭头函数中，它是固定的。

箭头函数this指向的固定，并不是因为箭头函数内部有绑定this的机制，而是因为箭头函数根本没有自己的this，因此内部的this就是外层代码块的this。正因为它没有this，所以不能用作构造函数。

除了this，一下三个变量在箭头函数中也是不存在的：arguments、super、new.target。

由于箭头函数没有自己的this，因此也不能使用call(),apply(),bind()这些方法去改变this的指向。

箭头函数也可以嵌套使用。

函数绑定

箭头函数简化了显示绑定this对象的写法（call，apply，bind）。但箭头函数并不适用于所有的场合，所以ES7提出了“函数绑定”运算符，用来取代call，apply，bind调用。

函数绑定运算符是并排的两个双冒号(::)，双冒号左边是一个对象，右边是一个函数。该运算符会自动将左边的对象，作为上下文环境（this对象），绑定到右边的函数上面。

如果双冒号左边为空，右边是一个对象的方法，则将该方法绑定在该对象上面。

双冒号运算符返回的是原对象，因此可以采用链式写法。

尾调用优化

尾调用概念

尾调用是函数式编程的一个重要概念，式指某个函数的最后一步是调用另一个函数。

尾调用不一定出现在函数最后一行，只要是最后一步即可。

尾调用优化

函数调用会在内存中形成一个调用记录，成为调用帧，保存调用位置和内部变量等信息。

调用栈：如果在函数A的内部调用函数B，那么在A的调用帧上方还会形成一个B的调用帧，等B运行结束，将结果返回给A，B的调用帧才会消失。如果B函数内部还调用了函数C，那还有一个C的调用帧。所有的调用帧就形成一个“调用栈”。

尾调用由于是函数的最后一步操作，所以不需要保留外层函数的调用帧，因此调用位置、内部变量等信息都不会再用到，只要直接用内层函数的调用帧取代外层函数的调用帧就可以了。

尾调用优化：只保留内层函数的调用帧。

只有不再用到外层函数的内部变量，内层函数的调用帧才会取代外层函数的调用帧，否则无法进行“尾调用优化”。

ECMAScript术语

类型（type）

ECMAScript规范定义数据的集合。

原始值（primitive value）

ECMAScript规范定义的未定义、空、布尔、数字、字符串类型之一的成员。

原始值代表语言实现的最底层的数据。

对象（object）

对象类型的成员

对象属性的集合，并有一个原型对象，原型可以是空值。

构造器（constructor）

创建和初始化对象的函数对象。

构造器的“prototype”属性值是一个原型对象，它用来实现继承和共享属性。

原型（prototype）

为其他对象提供共享属性的对象。

当构造器创建一个对象，为了解决对象的属性引用，该对象会隐式引用构造器的“prototype”属性，通过程序表达式constructor.prototype可以引用到构造器的“prototype”属性，并且添加到对象原型里的属性，会通过继承与所有共享此原型的对象共享。另外，可使用Object.create内置函数，通过明确指定原型来创建一个新对象。

原生对象（native object）

ECMAScript实现中，并非由宿主环境，而是完全由规范定义其语义的对象。

标准的原生对象由ECMAScript规范定义，一些原生对象是内置的，其他的可在ECMAScript程序执行过程中构建。

内置对象（built-in object）

由ECMAScript实现提供，独立于宿主环境的对象，ECMAScript程序开始执行时就存在。

标准的内置对象由ECMAScript规范定义，ECMAScript实现可以指定和定义其他的。所有内置对象都是原生对象，一个内置构造器是个内置对象，也是个构造器。

宿主对象（host object）

由宿主环境提供的对象，用于完善ECMAScript执行环境。

任何对象，不是原生对象就是宿主对象。

未定义值（undefined value）

说明一个变量没有被分配值的一个原始值。

未定义类型（undefined type）

拥有唯一值“未定义值”的类型。

空值（null value）

代表对象值为空的一个原始值。

空类型（null type）

拥有唯一值“空值”的类型。

布尔值（boolean value）

布尔类型的成员。

只有两个布尔值，true和false。

布尔类型（boolean type）

由原始值true和false组成的类型。

布尔对象（boolean object）

对象类型的成员，它是标准内置构造器Boolean的一个实例。

通过使用new表达式，以一个布尔值作为参数调用Boolean构造器来创建布尔对象。由此产生的对象包含一个值为此布尔值的内部属性。一个Boolean对象可以强制转换为布尔值。

字符串值（string value）

一个字符串值是字符串类型的成员。通过序列中的每个整数值代表UTF-16文本的单个16位单元。然而，对于其值，ECMAScript只要求必须是16位无符号整数，除此之外没有任何限制或要求。

所有可能的字符串值的集合。

对象类型的成员，它是标准内置构造器String的一个实例。

原始值，对应一个64位双精度二进制IEEE754值。

一个数字值是数字类型的成员，直接代表一个数字。

数字类型（number type）

所有可能的数字值的集合，包括特殊的“Not-a-Number”(NaN)值，正无穷，负无穷。

无穷（Infinity）

NaN

对象类型的成员，标准内置构造器Function的一个实例，并且可做为子程序被调用。

Webpack code splitting

发表于 2015-11-05

Webpack Code Splitting

对于大型应用来说，将所有的代码都打包到一个文件中一次加载不是很明智的做法，尤其是在某些代码块只需要在特定情况下加载的时候。Webpack可以将你的代码按需分割成“chunks”加载。其他的一些“bundlers”被成为“layers”、“rollups”或“fragments”。这个特性被成为代码分割（code splitting）。

这是一个可选的功能。你可以在代码中定义自己的分割点。Webpack的任务只关心依赖关（dependencies）系、输出文件（output files）和运行时的东西（runtime stuff）。

Code Splitting不只是提取共同的代码放到一个公共的文件中，更重要的功能是拆分代码，按需加载。这可以保证应用程序首次请求时下载的是最小的和所需求的文件。

Defining a split point

AMD和CommonJs都可以通过不同的方式按需加载代码。两个都支持分割点。

require.ensure (CommonJs)

require.ensure方法保证了当调用callback的时候，dependencies中的每一个dependency都可以异步加载。callback调用时需要require方法作为参数。

require.ensure只加载模块，并不执行它们。

require (AMD)

AMD规范定义了一个require的异步方法。

当require被调用的时候，所有的dependencies都被加载，callback被调用时依赖所有被加载的dependencies。

AMD require方法加载并执行这些modules，在webpack modules中从左向右执行；require方法可以不加回调函数。

Chunk content（chunk内容）

所有的dependencies都是在split point处生成新的chunk。Dependencies也都是递归添加。

如果将函数表达式（或绑定函数表达式）作为回调函数传递给split point，Webpack也会自动在这个函数表达式中通过所需的依赖（dependencies require）生成chunk文件。

Chunk optimization（chunk优化）

  • 如果两个chunks包含相同的modules， 它们会被合并成一个。这可能会导致chunks有多个parents
  • 如果一个module在一个chunk的所有parents中都是可用的，它应该从这个chunk中删除
  • 如果一个chunk包含了另一个chunk的所有modules，这个chunk应该被保留，因为它能满足多个chunks

Chunk loading（chunk加载）

根据目标（target）运行时逻辑的配置选项，加载chunk并添加到bundle中。对于web target chunks是通过JSONP加载。一个chunk只能加载一次，并行请求被合并成一个。运行时检测所加载的chunks是否满足多个chunks。

Chunk types

Entry chunk

一个entry chunk（入口块）包含运行时的多个modules。如果chunk包含模块0就在运行时执行它，如果没有，则会等待chunks中包含了模块0后在执行它（每次当chunk中包含module 0）。

Noraml chunk

normal chunk不包含runtime，它仅包含一组modules，结构取决于chunk的加载算法。JSONP模块被封装在jsonp的回调函数中。chunk还包含chunk id的一个列表。

Initial chunk(non-entry)（初始chunk，非入口）

一个initial chunk是一个normal chunk。唯一的区别是：优化（optimization）将其来说更重要，因为它要计算初次加载的时间（像entry chunks）。chunk type可结合使用CommonsChunkPlugin。

Split app and vendor code

把应用分成两个文件，如index.js和vendor.js，我们可以require这个vendor文件在vendor.js中。然后将这个名字传递给CommonsChunkPlugin。

这将会从应用程序的chunk中删除掉所有在vendor chunk中的modules。bundle.js文件将不会包含任何依赖关系，只包含应用程序代码。其他的都在vendor.bundle.js中。

在HTMl中，要先引入vendor.bundle.js，然后在引入bundle.js。

Multiple entry chunks（多入口chunks）

可以配置多个entry points，因此就会有多个entry chunks。运行时会包含这个entry chunk，但在运行时一个页面只能包含一个entry chunk（也有例外情况）。

Running multiple entry points（运行多个entry points）

随着运行时CommonsChunkPlugin被移到commons chunk，这个entry point现在是initial chunks。然而只有一个initial chunk，多个entry chunks已经被加载，这样暴露了一个页面上可以运行多个entry points的可能性。

Commons chunk

CommonsChunkPlugin插件可以把出现在多个entry chunks中的modules移到一个新的entry chunk（commons chunk）中，运行时也可以移到commons chunk中。它的意思就是说原来旧的entry chunks就是现在的initial chunks。请参考所有的list of plugin。

Optimization

一些优化的插件根据标准块进行合并。

  • LimitChunkCountPlugin
  • MinChunkSizePlugin
  • AggressiveMergingPlugin

Named chunks

require.ensure函数接收第三个参数，必须是个字符串。如果两个split point传递相同的字符串，那它们就使用相同的chunk。

require.include

require.include是Webpack的专用函数，用来在当前chunk中添加新的module，但不执行该module（该语句在bundle中会被移除）。

如果一个module有多个child chunks的话，require.include函数是比较有用的。require.include父级会包含这个module，chunk中的modules实例会在chunks中消失。

async函数就是将Generator函数的星号换成了async，将yield换成了await而已。

async函数对Generator函数有以下三点的改进。

  • 内置执行器。Generator函数的执行必须依靠执行器，所以才有了co模块，而async函数自带执行器。
  • 更好的语义。async和await比星号和yield语义清楚很多。
  • 更广的适用性。co模块约定，yield命令后面只能是Thunk函数或Promise对象，而async函数的await后面，可以是Promise对象和原始类型

async函数的实现

async函数的实现，就是将Generator函数和自动执行器包装在一个函数里。

async函数的用法

async函数返回一个promise对象，可以使用then方法添加回调函数。当函数执行时候，如遇到await就先返回，等到触发的异步操作完成，再接着执行函数体内后面的语句。

需要注意的几点

  • await后面的promise对象，运行结果可能是reject，所以将await放在try catch代码块中
  • await只能用在async函数中，如果在普通函数中，就会报错
  • 如果希望多个请求并发执行，使用Promise.all方法
